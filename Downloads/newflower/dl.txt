## Kotlin-Inject 기반 DI 설계서  
_“Claude Code가 디렉터리만 열어도 바로 그래프를 생성할 수 있는”_ 형태로 작성했습니다.

### 1. Gradle 설정

루트 settings.gradle.kts

```kotlin
pluginManagement {
    repositories { mavenCentral() }
    plugins {
        id("org.jetbrains.kotlin.multiplatform") version "2.0.0"
        id("com.squareup.inject.gradle")          version "0.7.0" // Kotlin-Inject
    }
}
```


공통 모듈 core/common/build.gradle.kts

```kotlin
plugins {
    kotlin("multiplatform")
    id("com.squareup.inject")           // KSP + symbol-processing
}

kotlin {
    android()                             // 타깃만 선언, 안드로이드 API 불러오지 않음
    iosX64(); iosArm64(); iosSimulatorArm64()

    sourceSets.commonMain.dependencies {
        implementation("com.squareup.inject:inject-runtime:0.7.0")
    }
}
```


> 🔑 Kotlin-Inject 는 **`@Inject` + KSP** 만 쓰므로  
> Android·iOS 패키지를 자동으로 끌어오지 않습니다.

### 2. 패키지 & 파일 위치

```
core/
 ├─ common/
 │   └─ di/
 │       ├─ Injection.kt          // @Inject 확장만 필요
 │       └─ RootComponent.kt      // @Component
 ├─ domain/
 │   └─ usecase/
 │       └─ GetDiaryUseCase.kt
 └─ data/
     └─ SqlDelightDiaryRepository.kt
```

### 3. 코어 코드 예시

#### 3-1. Repository

```kotlin
// core/data/SqlDelightDiaryRepository.kt
class SqlDelightDiaryRepository @Inject constructor(
    private val queries: DiaryDbQueries
) : DiaryRepository {
    override suspend fun findById(id: DiaryId) = /* … */
}
```

#### 3-2. Use-case

```kotlin
// core/domain/usecase/GetDiaryUseCase.kt
class GetDiaryUseCase @Inject constructor(
    private val repo: DiaryRepository
) {
    suspend operator fun invoke(id: DiaryId) = repo.findById(id)
}
```

#### 3-3. DI 그래프 루트

```kotlin
// core/common/di/RootComponent.kt
@Component
abstract class RootComponent {
    abstract val queries: DiaryDbQueries         // SQLDelight table adapter
    abstract val diaryRepo: DiaryRepository
    abstract val getDiary: GetDiaryUseCase
}
```

컴파일 후 KSP가 **`GeneratedRootComponent`** 클래스를 자동 생성합니다  
(패키지는 원본과 동일).

### 4. 플랫폼-모듈 부트스트랩

```kotlin
// platform/android/Bootstrap.kt
object Bootstrap {
    private val driverProvider = AndroidDatabaseDriverProvider(/* ctx */)

    // SQLDelight Query 객체 먼저 생성
    private val diaryDb = DiaryDb(driverProvider.createDriver(DiaryDb.Schema, Config.DB_NAME))

    // 그래프 주입
    val root: RootComponent = GeneratedRootComponent(
        diaryDb.diaryQueries            // RootComponent 생성자 파라미터
    )
}
```

iOS 쪽도 동일하게 `IosDatabaseDriverProvider` 만 교체하면 됩니다.

> -  **런타임 리플렉션 0 ** → 보일러플레이트 없이 성능 유지  
> -  _core_ 모듈은 `@Inject`, `@Component` 외에 DI 코드를 전혀 갖지 않음 ― 플랫폼 완전 중립.

### 5. 테스트 그래프 예시

```kotlin
// test/common/di/TestComponent.kt
@Component
abstract class TestComponent {
    abstract val diaryRepo: DiaryRepository

    companion object {
        fun create(fakeRepo: DiaryRepository): TestComponent =
            GeneratedTestComponent(fakeRepo)
    }
}
```

단위 테스트에서 `TestComponent.create(FakeRepo())` 로 주입해 Mock 교체 완료.

### 6. Detekt · 품질 규칙 체크포인트

| 규칙 | 코드 반영 |
|------|-----------|
| SRP / SOLID | 파일 1 책임 원칙 → Repository·UseCase·Component 분리 |
| 매직넘버 금지 | 모든 숫자·문자열은 `ColorPalette`·`Dimens`·`Config` 상수 사용 |
| 하드코딩 금지 | 위 상수만 사용, 문자열 중복은 Detekt `StringLiteralDuplication` |
| Generic Exception 금지 | `ExceptionUtil.safe { … }` 사용 |
| 플랫폼 의존성 0 | `android.*`, `UIKit.*` 는 오직 `platform-*` 모듈에만 존재 |
| 보일러 최소화 | `@Inject` 로 생성자 주입, 팩토리 자동 생성 |

### 7. “ 한 줄 메시지

> _“모든 DI 객체는 `@Inject` 로 생성자 주입, 최상위 `RootComponent` 는 컴파일-타임에 `GeneratedRootComponent` 로 변환됩니다. 코어 모듈은 표준 Kotlin + SQLDelight 외 의존성이 없으며, 플랫폼 모듈에서만 실제 드라이버·파일 API 를 바인딩합니다.”_
