## 📌 확정된 방향 요약

| 주제 | 결정 |
|------|------|
| Unit of Work 적용 범위 | **배치·복합 쓰기 작업 전용**. 일반 CRUD 는 레포지토리 내부 트랜잭션으로 처리 |
| 공통 상수 파일 위치 | `core/common/src/commonMain/kotlin/com/flower/common/`파일: `ColorPalette.kt`, `Dimens.kt`, `Config.kt` |
| CI 품질 게이트 | Detekt + `TooGenericExceptionCaught` 차단 rule를 **즉시 활성화** |

## 2. 예외 처리 모듈화

```kotlin
// core/common/.../ExceptionUtil.kt
package com.flower.common

import kotlinx.coroutines.CancellationException

object ExceptionUtil {

    suspend inline fun  runCatchingSuspend(
        crossinline block: suspend () -> T
    ): Result = try {
        Result.success(block())
    } catch (e: CancellationException) {      // 코루틴 취소 전파
        throw e
    } catch (e: Throwable) {                  // TooGenericExceptionCaught 피함
        Result.failure(e)
    }
}
```

*모든 레포지토리·유스케이스에서 `try-catch` 대신 `ExceptionUtil.runCatchingSuspend` 사용.*

## 3. Unit of Work 템플릿 (SQLDelight 전용)

```kotlin
// core/common/.../UnitOfWork.kt
package com.flower.common

interface UnitOfWork {
    suspend fun  run(block: suspend () -> R): Result
}

// core/diary-data/.../SqlDelightUnitOfWork.kt
package com.flower.diary.data

import com.flower.common.Config
import com.flower.common.UnitOfWork
import com.squareup.sqldelight.db.SqlDriver
import kotlinx.coroutines.runBlocking

class SqlDelightUnitOfWork(
    driver: SqlDriver
) : UnitOfWork {

    private val db = DiaryDb(driver)          // auto-generated by SQLDelight

    override suspend fun  run(block: suspend () -> R): Result =
        runCatching {
            db.transactionWithResult { runBlocking { block() } }
        }
}
```

### 활용 예

```kotlin
class DiaryBatchService(
    private val uow: UnitOfWork,
    private val diaryRepo: DiaryRepository,
    private val flowerRepo: FlowerRepository
) {
    suspend fun saveDiaryAndFlower(
        diary: Diary,
        flower: Flower
    ): Result = uow.run {
        diaryRepo.save(diary)
        flowerRepo.save(flower)
    }
}
```

## 4. Repository 스탠더드

```kotlin
class SqlDelightDiaryRepository(
    driver: SqlDriver
) : DiaryRepository {

    private val queries = DiaryDb(driver).diaryQueries

    override suspend fun findById(id: DiaryId): Result =
        ExceptionUtil.runCatchingSuspend {
            queries.selectById(id.value).executeAsOneOrNull()?.toDomain()
        }

    override suspend fun save(model: Diary): Result =
        ExceptionUtil.runCatchingSuspend {
            queries.insertOrReplace(model.toDb())
        }
}
```

*일반 CRUD 는 자체 트랜잭션 (`insertOrReplace`) 으로 처리하고,  
여러 테이블을 한꺼번에 갱신할 때만 `UnitOfWork` 사용.*

## 5. CI 품질 게이트 스크립트

```yaml
# .github/workflows/quality.yml
name: quality

on: [push, pull_request]

jobs:
  detekt:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: ./gradlew detekt

  generic-exception-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: |
          if grep -R --line-number "catch (\\s*Exception" $(git ls-files '*.kt'); then
            echo "::error ::Generic Exception caught"
            exit 1
          fi
```

## 6. 다음 작업 순서

1. **공통 상수 파일** 세 개 커밋 → 기존 매직넘버 치환  
2. **ExceptionUtil 도입 PR** → 레포지토리·유스케이스 `try-catch` 제거  
3. **SqlDelightUnitOfWork** 추가 → 배치/복합 저장 로직만 교체  
4. **CI 워크플로우 적용** → PR 기준선 생성  
5. 향후 신규 코드 생성 시 Detekt 위반·Generic Exception 사용이 포함되면 **CI 실패**로 즉시 알림

모든 예시는 **안드로이드·iOS 종속 코드가 전혀 없는** 순수 Kotlin 멀티플랫폼 구현입니다.  
추가 조정이 필요하거나 세부 모듈 경로가 달라질 경우 알려주시면 즉시 반영하겠습니다.
