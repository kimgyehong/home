# 완전 신규 프로젝트 뼈대 설계서  
_“Claude Code가 폴더만 열어도 전체 구도가 보이게”_  

## 0. 개요

목표  
1. **Android + iOS 동시 지원**  
2. **순수 Kotlin 멀티플랫폼**(KMP)로 도메인·데이터 100% 공유  
3. **SQLDelight** 단일 DB 레이어  
4. **경량 DI**(Koin) 또는 _수동 팩토리_  
5. **UI 전용 모듈**만 각 플랫폼 API 사용


### 5-3. Repository 구현

```kotlin
class SqlDelightDiaryRepository(
    driverProvider: DbDriverProvider
) : DiaryRepository {

    private val db = DiaryDb(
        driverProvider.provide(DiaryDb.Schema, "diary.db")
    )
    private val queries = db.diaryQueries

    override suspend fun findById(id: DiaryId): Diary? =
        queries.selectById(id.value).executeAsOneOrNull()?.toDomain()

    private fun Diary.toDomain() = Diary(
        id = DiaryId(id),
        title = title,
        content = content,
        createdAt = Instant.fromEpochMilliseconds(created_at)
    )
}
```

## 6. Feature 계층

```
feature/diary

```

```kotlin
class DiaryViewModel(
    private val getDiary: GetDiary,
    private val saveDiary: SaveDiary
) : CoroutineScope by MainScope() {

    private val _state = MutableStateFlow(DiaryState.Empty)
    val state: StateFlow = _state.asStateFlow()

    fun load(id: DiaryId) = launch {
        _state.value = DiaryState.Loading
        _state.value = DiaryState.Success(getDiary(id))
    }
}
```

## 7. DI (Koin 기준)


commonMain 모듈 선언

```kotlin
val dataModule = module {
    single { SqlDelightDiaryRepository(get()) }
}

val domainModule = module {
    factory { GetDiary(get()) }
    factory { SaveDiary(get()) }
}
```



platform-android 초기화 (Application-class)

```kotlin
class FlowerApp : Application() {
    override fun onCreate() {
        super.onCreate()
        startKoin {
            androidContext(this@FlowerApp)
            modules(
                module {  // platformModule
                    single { AndroidDbDriverProvider(this@FlowerApp) }
                },
                dataModule,
                domainModule
            )
        }
    }
}
```


iOS 쪽은 `KoinIOS.start()` + Swift wrapper.

### 8-2. iOS (SwiftUI)

```swift
import shared // KMP framework

@MainActor
final class DiaryObservable: ObservableObject {
    private let viewModel = DiaryViewModel(
        getDiary: Koin.instance().getGetDiaryUseCase(),
        saveDiary: Koin.instance().getSaveDiaryUseCase()
    )

    @Published var state = DiaryStateEmpty()

    init() {
        viewModel.state.subscribe { state in
            self.state = state
        }
    }
}

struct DiaryScreen: View {
    @StateObject private var holder = DiaryObservable()
    var body: some View {
        switch holder.state {
        case is DiaryStateLoading: ProgressView()
        case let success as DiaryStateSuccess: Text(success.diary.title)
        default: EmptyView()
        }
    }
}
```

## 9. 테스트 & CI

| 모듈         | 테스트 명령                                     |
|--------------|-------------------------------------------------|
| 공통/도메인  | `./gradlew :core:diary-domain:jvmTest`          |
| 데이터       | `./gradlew :core:diary-data:jvmTest`            |
| Android UI   | `./gradlew :ui:android:connectedDebugAndroidTest` |
| iOS          | `./gradlew :shared:linkDebugFrameworkIosX64` → Xcode XCTest |

GitHub Actions 예시:

```yaml
android:
  runs-on: ubuntu-latest
  steps:
    - uses: actions/checkout@v4
    - run: ./gradlew :app:android:assembleDebug

ios:
  runs-on: macos-14
  steps:
    - uses: actions/checkout@v4
    - run: ./gradlew :shared:assembleXCFramework
    - run: xcodebuild -scheme ui-ios -destination 'platform=iOS Simulator,name=iPhone 15'
```

## 10. 규칙 & 금기 사항

| ✔ 반드시 지키기 | ✖ 절대 금지 |
|-----------------|------------|
| 플랫폼 코드 → `platform-*` 모듈 한정 | `androidx.*`·`UIKit` 타입을 core/feature 모듈에 두는 행위 |
| `expect/actual` 로 자원 분리 | Hilt·Dagger 사용 (Android 전용) |
| `core-*` 는 순수 Kotlin only | 공통 모듈 안에 `android.*`, `swift.*` import |
| Repository 반환 = `suspend` + 널 혹은 Either | UI 레이어에 SQLDelight Query 노출 |

## 11. 착수 체크리스트

1. `./gradlew build` 가 두 플랫폼 모두 성공  
2. `DiaryDb` 스키마가 각 드라이버에서 생성 확인  
3. Android Activity, iOS SwiftUI 화면에서 `DiaryViewModel` 상태 출력  
4. GitHub Actions 두 잡 모두 초록  

위 네 가지가 통과되면 **프로젝트 골격 완성**입니다.  
이 문서만 그대로 복사-붙여넣기 후 디렉터리와 파일을 차례로 생성하면 _Claude Code_ 가 구조 전체를 즉시 인지할 수 있습니다.
