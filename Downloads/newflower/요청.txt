### 1. 핵심 품질 원칙 체크리스트 (빌드 단계에서 CI-Gate)

| 항목 | 필수 규칙 | 자동 검증 방법 |
|------|-----------|----------------|
| SRP · SOLID | 한 클래스/파일 하나의 책임, DIP/OCP 준수 | Detekt `LongClass`, `ComplexInterface`, 리뷰 체크 |
| Detekt 전 규칙 통과 | `./gradlew detekt` 경고 = 0 | GitHub Actions Fail fast |
| 매직 넘버·하드코딩 금지 | `ColorPalette`, `Dimens`, `Config` 상수만 사용 | Detekt `MagicNumber`, `StringLiteralDuplication` |
| Generic Exception 금지 | `catch (Exception)` 사용 금지 | CI grep + Detekt `TooGenericExceptionCaught` |
| 플랫폼 의존성 배제 | `androidx`, `UIKit` import ⇒ 코어 모듈 오류 | KMP 컴파일 + Detekt `ForbiddenImport` |
| 보일러플레이트 최소화 | data/inline/extension 적극 사용 | 리뷰 + 코드자동생성(SQLDelight/ktx) |

### 2. 최종 모듈 트리 (플랫폼 중립 ↔ 플랫폼 특화 완전 분리)

```
root/
├── core/
│   ├── common/          # Color, Dimens, Config, ExceptionUtil, UoW 인터페이스
│   ├── domain/          # 엔티티 · 값객체 · UseCase (순수 Kotlin)
│   └── data/            # SQLDelight DB, Repository, UnitOfWork 구현
├── feature/
│   ├── diary/           # ViewModel · Facade · MVI State
│   └── collection/      # 도감 전용 로직
├── platform/
│   ├── android/         # DbDriverProvider · FileStore actual
│   └── ios/             # 위와 동등한 iOS actual
├── ui/
│   ├── android/         # Compose·XML, Activity
│   └── ios/             # SwiftUI
└── app/
    ├── android/         # 런처, DI Bootstrap
    └── ios/             # (Xcode 빌드용)
```

> 의존성 흐름 : `ui-* ➜ feature-* ➜ core-domain ➜ core-data ➜ core-common`  
> `platform-*` 모듈은 오직 **core-common** 의 `expect` 인터페이스를 충족만 합니다.

### 4. Unit of Work & Repository 표준 템플릿

```kotlin
// core/common/UnitOfWork.kt
interface UnitOfWork {
    suspend fun  run(block: suspend () -> R): Result
}

// core/data/SqlDelightUnitOfWork.kt
class SqlDelightUnitOfWork(
    driverProvider: DatabaseDriverProvider
) : UnitOfWork {
    private val db = DiaryDb(
        driverProvider.createDriver(DiaryDb.Schema, Config.DB_NAME)
    )
    override suspend fun  run(block: suspend () -> R): Result =
        runCatching { db.transactionWithResult { runBlocking { block() } } }
}

// core/data/SqlDelightDiaryRepository.kt
class SqlDelightDiaryRepository(
    driverProvider: DatabaseDriverProvider
) : DiaryRepository {

    private val queries = DiaryDb(
        driverProvider.createDriver(DiaryDb.Schema, Config.DB_NAME)
    ).diaryQueries

    override suspend fun findById(id: DiaryId): Result =
        ExceptionUtil.runCatchingSuspend {
            queries.selectById(id.value).executeAsOneOrNull()?.toDomain()
        }

    override suspend fun save(model: Diary): Result =
        ExceptionUtil.runCatchingSuspend {
            queries.insertOrReplace(model.toDb())
        }
}
```

*일반 CRUD ⇒ Repository 내부 트랜잭션*  
*여러 테이블 ‘동시에’ 갱신 ⇒ `UnitOfWork.run { … }`* 호출

### 5. 예외 처리 모듈 (코루틴 취소 보존 + Generic Exception 회피)

```kotlin
object ExceptionUtil {
    suspend inline fun  runCatchingSuspend(
        crossinline block: suspend () -> T
    ): Result =
        try { Result.success(block()) }
        catch (e: CancellationException) { throw e }
        catch (e: Throwable)             { Result.failure(e) }
}
```

> 모든 `try-catch`를 **한 줄**로 통일 → 보일러플레이트·Detekt 위반 ZERO

### 9. 가이드 적용 순서 (실제 작업 TO-DO)

1. **ColorPalette / Dimens / Config** 파일 생성 후  
   기존 매직넘버 → 상수 치환 PR  
2. **ExceptionUtil** 통합 → 레포 내부 `try-catch` 일괄 제거  
3. **SqlDelightUnitOfWork** 추가 → 배치 저장 로직만 교체  
4. **CI 품질 게이트** 활성화 → 신규 위반 실시간 차단  
5. **플랫폼 actual 구현** (Driver·FileStore 등) ‑ 플랫폼 모듈에만 작성  
6. **UI 모듈** (Android · iOS) 은 각 네이티브 도구대로 개발  
   → 코어 계층으로 절대 의존성 역류 금지
